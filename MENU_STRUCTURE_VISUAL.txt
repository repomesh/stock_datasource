╔════════════════════════════════════════════════════════════════════════════════╗
║                    DATABASE SCHEMA FOR AGENT CONFIG                            ║
╚════════════════════════════════════════════════════════════════════════════════╝

CURRENT: File-based JSON storage
  Location: /data/workflows/{workflow_id}.json
  Limitation: Not queryable, poor audit trail

RECOMMENDED: ClickHouse table
  
  Table: agent_configurations
  
  Columns:
    - id (String) - UUID
    - name (String)
    - category (String) - "analysis", "comparison", "screening"
    - description (String)
    - system_prompt (String) - AI role definition
    - user_prompt_template (String) - {{var}} substitution
    - selected_tools (Array(String)) - MCP tool names
    - variables (String) - JSON array of variables
    - config_metadata (String) - JSON for extensibility
    - is_active (Boolean)
    - is_template (Boolean)
    - created_by (String)
    - created_at (DateTime)
    - updated_at (DateTime)
  
  Ordering: (category, created_at)
  Engine: MergeTree()
  
  BENEFITS:
    ✓ Full audit trail
    ✓ Query-able: Filter by category, creator, date, etc.
    ✓ Better scalability
    ✓ Integration with system_structured_logs
    ✓ Version control support
    ✓ Analytics on configuration usage

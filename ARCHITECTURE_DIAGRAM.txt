KEY PATTERNS:

1. REQUEST FLOW:
   Input → Orchestrator → Middleware (before) → Specialist Agent(s) 
        → Middleware (after) → SSE Stream → Frontend

2. CONCURRENCY:
   • Parallel: MarketAgent + ReportAgent (safe groups)
   • Sequential: Handoff via Command(goto=...)
   • Arena: N agents compete concurrently

3. MEMORY:
   • A: LangGraph MemorySaver (checkpoint)
   • B: Tool result compression
   • D: Shared state storage
   • E: Long-term preferences

4. EXTENSION:
   • Plugin system (30+ modules)
   • Skill registry (MCP + builtin + workspace)
   • Strategy registry (AI-generated + builtin)
   • Agent registry (explicit + scanning fallback)

EVOLUTION ROADMAP:

Phase 1 (In Progress): Unified Agent Runtime 
  → Consolidate OrchestratorAgent, WorkflowAgent, MultiAgentArena

Phase 2: Skill Registry Standardization
  → Unified skill/tool descriptor and registration

Phase 3: SubAgent Protocol
  → Better sub-task isolation and coordination

Phase 4: Resource Optimization
  → Reduce redundant classification, context building, metadata
